# 代码审查报告 - Container层

## 项目概述

jxlsb 是纯Java实现的XLSB格式读写库，零依赖，采用堆外内存架构。Container层负责ZIP容器的读写管理。

## 优秀设计

1. **AutoCloseable实现**：`XlsbContainer`和`XlsbContainerReader`均实现`AutoCloseable`，支持try-with-resources语法
2. **缓冲优化**：`XlsbContainer`使用256KB缓冲区减少磁盘IO
3. **工具类模式**：`RelsGenerator`采用final class + 私有构造函数，符合规范
4. **不可变数据类**：`SheetInfo`字段均为final，线程安全
5. **EntryMap缓存**：`XlsbContainerReader`预先构建entry映射，避免重复遍历

## 详细分析

### 资源泄漏分析

#### XlsbContainerReader.getEntryBytes() 资源泄漏

```java
// 当前实现
public byte[] getEntryBytes(String name) throws IOException {
    ZipEntry entry = entries.get(name);
    if (entry == null) {
        return null;
    }
    InputStream in = zipFile.getInputStream(entry);  // 未关闭
    ByteArrayOutputStream out = new ByteArrayOutputStream();
    byte[] buffer = new byte[8192];
    int len;
    while ((len = in.read(buffer)) != -1) {
        out.write(buffer, 0, len);
    }
    in.close();  // 正常流程关闭
    return out.toByteArray();
}
```

**问题**：如果`in.read()`或`out.write()`抛出异常，`in.close()`不会执行。

**建议修改**：
```java
try (InputStream in = zipFile.getInputStream(entry)) {
    ByteArrayOutputStream out = new ByteArrayOutputStream();
    byte[] buffer = new byte[8192];
    int len;
    while ((len = in.read(buffer)) != -1) {
        out.write(buffer, 0, len);
    }
    return out.toByteArray();
}
```

#### XlsbContainer.close() 异常处理

```java
// 当前实现
@Override
public void close() throws IOException {
    try {
        zipOut.finish();
    } finally {
        zipOut.close();   // 若此处抛异常
        fileOut.close();  // 此行不执行
    }
}
```

**建议修改**：
```java
@Override
public void close() throws IOException {
    try {
        zipOut.finish();
    } finally {
        try {
            zipOut.close();
        } finally {
            fileOut.close();
        }
    }
}
```

## 总结

Container层整体设计清晰，职责划分合理。主要问题集中在资源管理方面：

1. **必须修复**：`getEntryBytes()`资源泄漏、异常处理空白
2. **应当改进**：`close()`方法的异常安全、方法文档补充
3. **建议优化**：StringBuilder预分配、代码可读性提升
